# 활용신청한 API를 pchahub에 등록하는 방법

data.go.kr에서 활용신청 승인된 API의 **데이터명 + Endpoint URL** 2가지만
알려주시면 됩니다. 키나 사용자 인증키는 채팅에 붙이지 마세요.

## 가장 빠른 입력 형식 (복사·붙여넣기용)

다음 형식으로 채팅에 붙여주시면 자동으로 `registry.ts`에 추가합니다.

```
이름: 공정거래위원회_가맹정보_업종별 브랜드변동현황 제공 서비스
URL: https://apis.data.go.kr/1130000/FftcIndutyBrandStatsService
포맷: JSON+XML

이름: 공정거래위원회_페어데이터_브랜드별 가맹점/직영점 집계 ...
URL: https://apis.data.go.kr/1130000/...
포맷: JSON

(이런 식으로 N개)
```

## 또는 캡처/스크린샷도 OK

각 API의 "개발계정 상세보기" 페이지 캡처를 보내주시면 OCR로 읽어서 정리합니다.
캡처에 노출된 키는 자동으로 무시하고 (보안), 데이터명과 Endpoint만 추출합니다.

## 우선순위 (작업 순서 권장)

이미 받으신 96개 중 **프랜차이즈 가맹 관련만** 우선 정리. pchahub 가치
순서대로:

| 우선 | API명 키워드 | 가치 |
|---|---|---|
| 🔴 1 | "페어데이터" + "가맹점/직영점 집계" | 매장 수·평균매출 (정제 데이터) |
| 🔴 2 | "브랜드 목록 정보" | 모든 가맹 브랜드 디렉토리 |
| 🔴 3 | "정보공개서 본문 조회" | 가맹비·보증금 원본 (XML) |
| 🟡 4 | "가맹본부 일반 정보" | 회사 메타 |
| 🟡 5 | "업종별 브랜드변동현황" ← 받으심 ✅ | 카테고리 트렌드 |
| 🟡 6 | "지역별 업종별 평균 매출액" | 매물 페이지 옆 시세 |
| 🟢 7 | "가맹본부 등록 목록" | 사업자번호 (검증용) |
| 🟢 8 | "가맹본부 재무정보" | 기업 신뢰도 |
| 🟢 9 | "정보공개서 목차/Viewer" | 원본 PDF 링크 |

## 한 번에 다 처리하는 방법

96개 중 위 9개만 골라서 보내주시면 pchahub 핵심은 끝납니다. 나머지는:
- 부동산 통계 → themyungdang으로 분리
- 소상공인·국세청 → bestplace나 다른 사이트로 분리
- 가맹과 무관한 것 → 일단 보류

활용신청한 96개의 데이터명만 단순 리스트로 붙여주셔도(엑셀 한 컬럼 복붙
형태), 제가 자동으로 가맹 관련만 추리고 우선순위 매겨드립니다.
